# Локальный запуск

Этот раздел поможет запустить Bedrock локально с нуля.

## 1. Требования

- Bun (менеджер пакетов и запуск скриптов)
- Node.js 24.x (runtime приложений)
- Docker (PostgreSQL и TigerBeetle)

## 2. Поднимите инфраструктуру

```bash
docker compose -f infra/docker-compose.yml up -d
```

Поднимутся сервисы:

- PostgreSQL: `localhost:5432`
- TigerBeetle: `localhost:3000`

Проверка контейнеров:

```bash
docker compose -f infra/docker-compose.yml ps
```

## 3. Проверьте `.env`

Минимальная конфигурация:

```bash
DB_HOST=localhost
DB_PORT=5432
DB_NAME=postgres
DB_USER=postgres
DB_PASSWORD=postgres
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/postgres
TB_ADDRESS=127.0.0.1:3000
TB_CLUSTER_ID=1
BETTER_AUTH_URL=http://localhost:3002
BETTER_AUTH_TRUSTED_ORIGINS=http://localhost:3001
PORT=3002
```

## 4. Установите зависимости

```bash
bun install
```

## 5. Запустите приложения

API:

```bash
bun run --filter=api dev
```

Web:

```bash
bun run --cwd apps/web dev
```

Workers:

```bash
bun run --filter=workers worker:all
```

Docs:

```bash
bun run --cwd apps/docs dev
```

## 6. Быстрая проверка API

Проверка OpenAPI:

```bash
curl -i http://localhost:3002/api/open-api
```

Проверка worker-мониторинга:

```bash
curl -i http://localhost:8081/health
curl -i http://localhost:8081/metrics
```

## 7. Частые ошибки на старте

- Порт занят (`3000/3001/3002/3003/8081`): завершите конфликтующий процесс.
- Ошибка подключения к БД: проверьте `DATABASE_URL` и состояние контейнера Postgres.
- Ошибка TigerBeetle: проверьте `TB_ADDRESS` и что контейнер слушает `3000`.
