# Структура репозитория

Bedrock использует Turborepo и разделяет код на приложения и пакеты.

## Workspace-структура

- `apps/*`
- `packages/platform/*`
- `packages/modules/*`
- `packages/packs/*`
- `packages/sdk/*`
- `packages/tooling/*`

## Что где находится

- `apps/api` - HTTP API-адаптер (Hono)
- `apps/web` - интерфейс пользователя (Next.js)
- `apps/workers` - фоновые циклы обработки
- `apps/docs` - документация (Nextra)

- `packages/platform/*` - базовые runtime-подсистемы (ledger, documents, connectors, orchestration)
- `packages/modules/*` - доменные модули (payments, fx, fees, customers...)
- `packages/packs/*` - бухгалтерские пакеты/шаблоны

## Направление зависимостей

```text
apps/* -> packages/modules/* -> packages/platform/* -> packages/packs/*
```

Это главное правило архитектуры: внешние слои зависят от внутренних, но не наоборот.

## Стиль сервисов

Сервисы в Bedrock пишутся фабриками-замыканиями, не классами.

```ts
export function createPaymentsService(deps: PaymentsServiceDeps) {
  const context = createPaymentsServiceContext(deps);

  async function createDraft(input: CreatePaymentIntentInput) {
    // validate -> db transaction -> emit events
  }

  async function post(input: PostPaymentInput) {
    // accounting + connector intent + orchestration route
  }

  return { createDraft, post };
}
```

Плюсы такого подхода:

- проще тестировать зависимости
- проще дробить большие сервисы на handlers
- меньше скрытого состояния

## Как читать код новичку

1. Начните с `apps/api/src/composition/*`.
2. Посмотрите `apps/api/src/modules/registry.ts`.
3. Перейдите в конкретный модуль в `packages/modules/*`.
4. Если нужна механика ядра - в `packages/platform/*`.
