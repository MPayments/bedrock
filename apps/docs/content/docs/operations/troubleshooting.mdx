# Диагностика

Ниже самые частые проблемы и короткие шаги проверки.

## 1. Приложение не стартует (port busy)

Проверьте стандартные порты:

- `3000` TigerBeetle
- `3001` Web
- `3002` API
- `3003` Docs
- `8081` Workers monitoring

Пример проверки:

```bash
lsof -i :3002
```

## 2. API отвечает 401/403

Проверьте:

- есть валидная сессия (cookie)
- пользователь имеет нужные permissions
- `BETTER_AUTH_URL` и `BETTER_AUTH_TRUSTED_ORIGINS` согласованы

## 3. Падает подключение к PostgreSQL

Проверьте:

```bash
docker compose -f infra/docker-compose.yml ps
```

И значения:

- `DATABASE_URL`
- `DB_HOST`, `DB_PORT`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`

## 4. Не работает ledger-поток

Проверьте:

- TigerBeetle контейнер запущен
- `TB_ADDRESS` доступен
- `TB_CLUSTER_ID` корректен

## 5. Воркеры в состоянии degraded

Проверьте `/health` и `/metrics`:

```bash
curl http://localhost:8081/health
curl http://localhost:8081/metrics
```

Ищите:

- последние ошибки циклов
- рост `bedrock_worker_errors_total`

## 6. Docs не собирается

Проверьте:

- `bun install` после изменения зависимостей
- корректность `_meta.ts`
- отсутствие битых ссылок на страницы

Затем:

```bash
bun run --cwd apps/docs check-types
bun run --cwd apps/docs build
```
