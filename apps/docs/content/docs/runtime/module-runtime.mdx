# Управление модулями

Bedrock поддерживает runtime-управление компонентами через `@bedrock/component-runtime`.

Это значит, что компонент можно отключить без изменения кода и релиза.

## Приоритет состояния

Эффективное состояние компонента вычисляется так:

1. override на уровне book
2. global override
3. значение по умолчанию из manifest

## Основные endpoints

- `GET /v1/system/components`
- `GET /v1/system/components/:componentId/effective`
- `PUT /v1/system/components/:componentId/state`
- `GET /v1/system/components/events`
- `GET /v1/system/components/runtime`

## Пример: посмотреть эффективное состояние компонента

```bash
curl -X GET http://localhost:3002/v1/system/components/payments/effective \
  -H 'Cookie: <session-cookie>'
```

## Пример: отключить компонент глобально

```bash
curl -X PUT http://localhost:3002/v1/system/components/payments/state \
  -H 'Content-Type: application/json' \
  -H 'Cookie: <session-cookie>' \
  -d '{
    "scope": "global",
    "enabled": false,
    "reason": "maintenance"
  }'
```

Когда компонент отключен, защищенные пути отдают `503` с признаком `MODULE_DISABLED`.
