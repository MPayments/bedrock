# Модель хранения

Bedrock использует:

- PostgreSQL как основной источник данных
- TigerBeetle для ledger-исполнения

## Семейства таблиц в PostgreSQL

- Auth: `user`, `session`, `account`, `verification`
- Documents: `documents`, `document_events`, `document_links`, `document_operations`
- Accounting/Ledger: `ledger_operations`, `postings`, `outbox`, `book_account_instances`
- Balances: `balance_positions`, `balance_holds`, `balance_events`
- Connectors: intents, attempts, events, references, cursors
- Orchestration: rules, corridors, limits, fee schedules, overrides
- Reconciliation: records, runs, matches, exceptions

## Почему так разделено

- проще масштабировать ответственность по пакетам
- проще вводить индексы под конкретный workload
- проще аудитить изменения по доменам

## Полезные проверки SQL

Пример: быстро посмотреть попытки платежа (идея запроса):

```sql
select intent_id, attempt_no, status, created_at
from payment_attempts
where intent_id = '<intent-id>'
order by attempt_no asc;
```

Пример: проверить событие webhook по ключу идемпотентности:

```sql
select provider_code, webhook_idempotency_key, created_at
from connector_events
where webhook_idempotency_key = '<key>';
```
