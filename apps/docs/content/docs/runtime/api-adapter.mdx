# API-адаптер

`apps/api` - входная точка Bedrock для внешних клиентов.

## Что делает API-слой

- аутентифицирует пользователя
- проверяет permissions
- валидирует payload
- вызывает сервисы модулей
- отдает единый JSON-контракт

## Ключевые маршруты `/v1`

- `/v1/payments`
- `/v1/connectors`
- `/v1/orchestration`
- `/v1/reconciliation`
- `/v1/system/components`
- и другие прикладные модули (`accounts`, `customers`, `currencies`...)

## Важные технические endpoints

- OpenAPI: `/api/open-api`
- Scalar UI: `/docs`

## Пример: проверить OpenAPI

```bash
curl -s http://localhost:3002/api/open-api | head -n 20
```

## Пример: получить список провайдеров connectors

```bash
curl -X GET http://localhost:3002/v1/connectors/providers \
  -H 'Cookie: <session-cookie>'
```

## Идемпотентность

Для мутаций (кроме create-операций, где ключ не нужен по контракту) используйте `Idempotency-Key`.

```bash
curl -X POST http://localhost:3002/v1/payments/<id>/approve \
  -H 'Cookie: <session-cookie>' \
  -H 'Idempotency-Key: payment-approve-<id>-v1'
```

Это защищает от двойного выполнения при ретраях клиента.
